## PHASE B — LIVE TRANSLATOR (walkie-talkie mode)

Add a new screen called "Live Translator" accessible from:
- Bottom navigation as a 5th tab with icon 🔄
- A card on the home screen: "Speak any language — hear it in another"

The screen layout:
- Full screen divided into two halves by a horizontal divider
- Top half: Person A (e.g. parent who speaks Amharic)
  - Language selector: pill buttons [Amharic] [Oromo] [English]
  - Large press-and-hold button: "Hold to speak"
  - Transcript area: shows what was heard in light text
  - Translation area: shows translated text in bold
- Bottom half: Person B (e.g. teacher who speaks Oromo)
  - Same layout, different language selection
  - Large press-and-hold button at the bottom half
- Between the two halves: a swap button (⇅) that switches A and B languages

How it works (implement this exactly):

1. User presses and holds the speak button
2. Use Web Speech API SpeechRecognition:
   - lang attribute: 'am-ET' for Amharic, 'om' for Oromo, 'en-US' for English
   - continuous: false, interimResults: true
3. While speaking: show animated waveform bars (3–5 bars with staggered CSS animation)
4. On recognition result:
   a. FIRST — look up exact word match in your local words array. If found, use your stored translation immediately (instant, no API call)
   b. If not found — call LibreTranslate API (free, open-source):
      POST https://libretranslate.com/translate
      Body: { q: recognizedText, source: sourceLangCode, target: targetLangCode, api_key: '' }
      Language codes: Amharic = 'am', Oromo = 'om', English = 'en'
      Note: LibreTranslate may not support Amharic/Oromo — if API returns error, fall back to showing original text with a note "Full translation coming soon"
   c. Display translated text in the other person's half
5. Automatically play translated text using SpeechSynthesis:
   - lang: 'am-ET' for Amharic, 'en-US' for English
   - pitch: 1, rate: 0.85 (slightly slower for clarity)
   - Show speaker animation while playing

Situations mode (add below the translator):
- A scrollable row of situation cards: "Market 🛒" | "Bus Station 🚌" | "Health Center 🏥" | "Meeting Elders 🙏" | "School Meeting 🏫" | "Coffee Ceremony ☕"
- Tapping a situation loads 5 pre-written phrase pairs (Amharic ↔ Oromo ↔ English) relevant to that situation
- Each phrase has a tap-to-speak button that speaks it in the selected language
- These phrases come from your local words/phrases data — no API needed

export function getCurriculumContext(gradeLevel, language) {
  const grade = CURRICULUM['grade' + gradeLevel];
  if (!grade) return null;
  return grade[language] || grade['amharic'] || null;
}

export function findTopicForQuestion(questionText, gradeLevel, language) {
  const ctx = getCurriculumContext(gradeLevel, language);
  if (!ctx) return null;
  const lower = questionText.toLowerCase();
  for (const topic of ctx.topics) {
    if (topic.keywords.some(kw => lower.includes(kw))) {
      return topic;
    }
  }
  return null;
}

## PHASE F — SUPABASE REAL CONNECTION

Connect the app to real Supabase. Create a Supabase project at supabase.com (free tier) and:

1. Environment variables needed (.env file):
   VITE_SUPABASE_URL=your_project_url
   VITE_SUPABASE_ANON_KEY=your_anon_key
   VITE_GOOGLE_VISION_KEY=your_vision_api_key (for camera feature)
   VITE_HASAB_API_KEY=your_hasab_key (when available)

2. Create Supabase client in src/lib/supabase.js:
   import { createClient } from '@supabase/supabase-js'
   export const supabase = createClient(import.meta.env.VITE_SUPABASE_URL, import.meta.env.VITE_SUPABASE_ANON_KEY)

3. Migrate local words array to Supabase words table. Keep local array as offline fallback — on app load, try to fetch from Supabase; if offline, use local array.

4. Sessions: when a practice session completes, save to Supabase sessions table (user_id, child_id, session_type, words_practiced, score, duration_seconds).

5. Children: save child profiles to Supabase children table. Link to auth user.

6. Real-time parent dashboard: use Supabase Realtime to subscribe to sessions table for the active child:
   supabase.channel('child-sessions').on('postgres_changes', { event: 'INSERT', schema: 'public', table: 'sessions', filter: 'child_id=eq.' + childId }, payload => { refreshDashboard(); }).subscribe()
